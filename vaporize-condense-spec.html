Concept

Vaporize & Condense uses "particle" metaphor to communicate that the old UI state disperses into thin air while a fresh set of options condenses from nothing. The two phases run in sequence — vaporise fires immediately on tap, condense fires once the old chip has vanished (~300 ms). Only suggestion chips participate in the animation; the selected (primary) chips are stable.

CSS Reference
/* Easing token */
--fluid: cubic-bezier(0.16, 1, 0.3, 1);

/* Phase 1 — vaporize (applied on tap) */
.chip.vaporize {
  opacity: 0;
  transform: scale(1.5);
  filter: blur(4px);
  transition: opacity 300ms var(--fluid),
              transform 300ms var(--fluid),
              filter 300ms var(--fluid);
}

/* Phase 3 — condense (new chips enter) */
@keyframes condenseAnim {
  from { opacity: 0; transform: scale(0.5); }
  to   { opacity: 1; transform: scale(1);   }
}
.chip.condense {
  animation: condenseAnim 400ms var(--fluid) both;
  /* animation-delay set inline: index * 40ms */
}

/* Deselect exit */
.chip {
  transition: all 200ms var(--fluid);
}
/* JS sets: transform: scale(0.8), opacity: 0
   then after 200ms swaps class to .suggested */
